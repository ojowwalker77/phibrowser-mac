# Fork releases

This fork produces Apple Silicon builds under the bundle identifier
`com.ojowwalker77.PhiBrowser`. The app is signed with Jonatas Walker's
Developer ID certificate, notarized by Apple, published as ZIP and DMG assets,
and updated through the fork's own Sparkle appcast.

## Publish a version

1. Merge the desired changes into `main`.
2. Open **Actions → Release Fork → Run workflow**.
3. Enter a stable version such as `0.1.0`.
4. Wait for the signing, notarization, and release job to finish.
5. Download the DMG from the new GitHub Release.

Pushing a stable tag such as `v0.1.0` runs the same pipeline. Do not publish the
same version twice: Sparkle uses the numeric build number generated by Actions
to order updates.

## Update the Chromium framework

Change the repository variable `PHI_FRAMEWORK_VERSION` to a published
`phibrowser/phibrowser-framework` tag. The download script refuses assets whose
SHA-256 does not match the digest published by GitHub.

## Required repository configuration

The release workflow expects these Actions secrets:

- `MACOS_CERTIFICATE_P12`
- `MACOS_CERTIFICATE_PASSWORD`
- `APPLE_API_KEY`
- `APPLE_API_KEY_ID`
- `APPLE_API_ISSUER`
- `SPARKLE_PRIVATE_KEY`

It also expects `APPLE_TEAM_ID` and `PHI_FRAMEWORK_VERSION` as Actions
variables. Private key backups belong outside the repository.
